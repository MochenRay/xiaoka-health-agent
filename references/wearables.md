## 核心指标详解

### HRV（心率变异性）

**什么是HRV：** 心跳之间时间间隔的变化程度。反映自主神经系统平衡。

| HRV水平 | 含义 |
|---------|------|
| 高（个人基线以上） | 恢复良好、副交感神经主导、可以训练 |
| 低（个人基线以下） | 压力/疲劳/恢复不足、建议轻练或休息 |
| 持续走低 | 过度训练、生病前兆、睡眠差 |

**重要原则：**
- HRV是**个体指标**，不跟别人比，跟自己的基线比
- 年龄越大HRV越低（正常）
- 测量条件要一致（建议晨起或睡眠中测）
- 单日波动正常，看**7天趋势**
- 28岁男性参考范围：40-100ms（RMSSD），但个体差异大

**影响HRV的因素：**
| 提高HRV | 降低HRV |
|---------|---------|
| 充足睡眠 | 睡眠不足 |
| 规律有氧运动 | 过度训练 |
| 冥想/深呼吸 | 酒精 |
| 良好营养 | 高压力 |
| 水分充足 | 脱水 |
| 适度恢复日 | 生病/感染 |

### 静息心率（RHR）

| RHR范围 | 评估 |
|---------|------|
| <50 bpm | 优秀（运动员水平） |
| 50-60 bpm | 很好 |
| 60-70 bpm | 正常 |
| 70-80 bpm | 一般（久坐者常见） |
| >80 bpm | 偏高，需改善 |

**趋势比绝对值更重要：** RHR持续下降说明心肺功能在改善。

### 血氧（SpO2）

| 水平 | 含义 |
|------|------|
| 95-100% | 正常 |
| 90-94% | 偏低，关注 |
| <90% | 就医 |

**睡眠中SpO2反复下降** → 可能提示睡眠呼吸暂停，建议做睡眠监测。

### 睡眠分期

| 阶段 | 占比目标 | 功能 |
|------|---------|------|
| 深睡（N3） | 15-25% | 身体修复、生长激素分泌、肌肉恢复 |
| REM | 20-25% | 记忆巩固、情绪调节 |
| 浅睡（N1+N2） | 50-60% | 过渡阶段 |
| 清醒 | <5% | 正常有少量醒来 |

**提高深睡：** 规律运动、避免睡前酒精、卧室温度18-20°C、固定作息
**提高REM：** 充足总睡眠时长（REM主要在后半夜）、减少酒精

### 训练负荷 / Strain

**Whoop Strain量表（0-21）：**
| Strain | 强度 | 示例 |
|--------|------|------|
| 0-9 | 轻度 | 日常活动、散步 |
| 10-13 | 中度 | 快走、轻度力量训练 |
| 14-17 | 高强度 | 跑步、游泳、HIIT |
| 18-21 | 极限 | 比赛、长距离耐力 |

**Strain与恢复的关系：**
- 恢复度高（绿色/67%+）→ 可以高Strain训练
- 恢复度中（黄色/34-66%）→ 中等Strain
- 恢复度低（红色/<34%）→ 轻度活动或休息

### 体表温度

- 可穿戴设备测的是**皮肤温度**，不是核心体温
- 看**偏差趋势**，不看绝对值
- 持续偏高 → 可能生病、月经周期（女性）、过度训练
- Oura Ring的温度追踪最准确（手指血管丰富）

## Whoop 深度指南（Yi即将购买）

### 核心功能

1. **Recovery（恢复度）** — 每天0-100%评分
   - 基于：HRV、RHR、呼吸频率、睡眠表现
   - 绿色(67%+)：全力训练 | 黄色(34-66%)：适度 | 红色(<34%)：休息

2. **Strain（训练负荷）** — 每天0-21评分
   - 基于心率在各区间的时间
   - 目标：匹配恢复度来决定训练强度

3. **Sleep（睡眠）** — 睡眠表现评分
   - 追踪：总时长、各阶段占比、呼吸频率、血氧
   - **Sleep Need（睡眠需求）**：根据前一天Strain和恢复状态计算你需要多少睡眠
   - **Sleep Debt（睡眠债务）**：累计不足

4. **Journal（日志）** — 行为与数据关联
   - 记录：咖啡因、酒精、补剂、压力、屏幕时间等
   - Whoop会分析哪些行为影响你的恢复和睡眠
   - **这个功能对Yi很有价值** — 量化哪些习惯有效

5. **Health Monitor** — 长期健康趋势
   - 呼吸频率异常检测
   - 静息心率趋势
   - HRV基线变化

### Whoop 5.0 会员档位（2026）

| 档位 | 价格 | 定位 | 关键差异 |
|------|------|------|---------|
| **One** | $149/年起 | 入门健身 | 基础Recovery/Strain/Sleep，无Journal、无压力监测 |
| **Peak** | $239/年 | 进阶健康+健身 | +实时压力监测、Journal行为关联、高级睡眠分析、HSA/FSA可报销 |
| **Life** | 更高（未公开） | 医疗级 | +ECG、血压监测、医疗级健康监测功能 |

**推荐Peak** — 性价比最好，Journal和压力监测是核心功能。One太基础，Life对健康年轻人没必要。

### Whoop 佩戴建议
- **默认：** 手腕（非惯用手）
- **替代：** 二头肌带（Whoop Body系列）— 更准确
- **运动时：** 确保贴紧皮肤，出汗时可能松动
- **游泳：** 防水50米，可以戴着游（Yi的优势运动）
- **充电：** 滑入式充电，不需要摘下来

### Whoop数据解读 — 减脂期特别关注

| 指标 | 关注点 |
|------|--------|
| HRV趋势 | 如果持续下降 → 热量缺口可能太大或训练过度 |
| RHR趋势 | 应该随减脂逐渐下降；如果上升 → 过度训练/压力 |
| 恢复度 | 连续3天红色 → 需要减训练量或增加热量 |
| 睡眠表现 | 目标≥85%；低于70%影响减脂效率 |
| Strain目标 | 减脂期不需要极端Strain，10-14日均就够 |

## 可穿戴设备常见误区

| 误区 | 事实 |
|------|------|
| "燃脂心率区间最减脂" | 总热量消耗更重要，不要执着Zone 2 |
| "设备显示的卡路里消耗很准" | 误差通常20-40%，**不要用设备消耗来决定吃多少** |
| "HRV越高越好" | 个体基线不同，看趋势不看绝对值 |
| "深睡占比低=睡眠差" | 设备分期不100%准确，整体趋势更重要 |
| "每天数据都要完美" | 看周均值，单日波动正常 |
| "戴了设备就会自动变健康" | 设备是工具，行动才是关键 |

## 数据驱动的训练调整

### 基于Whoop恢复度的训练决策

```
恢复度 ≥ 67%（绿色）
  → 今天可以上强度
  → 力量训练：正常重量、正常组数
  → 有氧：可以做间歇/高强度
  → 目标Strain：14-18

恢复度 34-66%（黄色）
  → 适度训练
  → 力量训练：减重量10-20%或减组数
  → 有氧：中等强度稳态
  → 目标Strain：10-14

恢复度 < 34%（红色）
  → 主动恢复
  → 散步、轻度拉伸、瑜伽
  → 不做高强度训练
  → 目标Strain：<10
  → 关注睡眠和营养
```

### 减脂期的设备数据红旗 🚩

| 信号 | 含义 | 行动 |
|------|------|------|
| HRV连续7天下降 | 身体压力过大 | 减少训练量或增100-200卡热量 |
| RHR连续上升 | 恢复不足/过度训练 | 增加休息日 |
| 睡眠表现连续<70% | 睡眠债务累积 | 优先改善睡眠 |
| 恢复度连续3天红色 | 系统性疲劳 | 安排deload周 |
| 体温持续偏高 | 可能生病 | 减训练、监测症状 |

## Whoop Cache 集成

如果你通过 Whoop API 缓存数据到本地 JSON 文件，可以直接读取解析。

### 推荐 Cache JSON 结构

```json
{
  "updated_at": "ISO timestamp",
  "recovery": {
    "score": 0-100,        // 恢复评分（绿>67, 黄34-66, 红<34）
    "hrv": float,           // 心率变异性 (ms, RMSSD)
    "rhr": float,           // 静息心率 (bpm)
    "spo2": float,          // 血氧 (%)
    "skin_temp": float,     // 皮肤温度 (°C)
    "calibrating": bool     // 是否还在校准期
  },
  "sleep": {
    "start": "UTC ISO",     // 入睡时间（UTC，需转本地时区）
    "end": "UTC ISO",       // 醒来时间（UTC，需转本地时区）
    "nap": bool,            // 是否午睡
    "performance": 0-100,   // 睡眠表现评分
    "efficiency": 0-100,    // 睡眠效率 (%)
    "total_min": int,       // 总睡眠时长（分钟）
    "rem_min": int,         // REM 睡眠（分钟）
    "deep_min": int,        // 深睡（分钟）
    "light_min": int,       // 浅睡（分钟）
    "awake_min": int,       // 清醒（分钟）
    "disturbances": int|null // 睡眠干扰次数
  },
  "cycle": {
    "strain": 0-21,         // 当日训练负荷
    "avg_hr": int,          // 平均心率 (bpm)
    "max_hr": int,          // 最大心率 (bpm)
    "kilojoule": float      // 消耗能量 (kJ)
  }
}
```

### 通用红线参考

| 指标 | 一般红线 | 建议行动 |
|------|---------|---------|
| Recovery | <30% 连续3天 | 强制休息/deload |
| HRV | 持续低于个人基线30%+ | 排查过训/压力/睡眠 |
| RHR | 持续高于个人基线15%+ | 增加休息、排查原因 |
| SpO2 | <95% | 关注呼吸系统状况 |
| Sleep performance | <50% 连续3天 | 干预作息 |
| Strain | >18 非比赛日 | 过训风险 |

### 使用注意
- sleep.start/end 是 **UTC 时间**，展示给用户时需转本地时区
- Whoop 校准期（calibrating=true）数据不准，前几天数据仅供参考
- 建议在 TOOLS.md 中配置 cache 文件路径和刷新频率等个人信息
