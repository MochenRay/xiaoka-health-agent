# 健康目标

> 首次使用时由小卡根据 Profile 自动生成。可随时通过对话更新（"我想调整目标"）。

## 主要目标

- **目标类型：** (必填) 减脂 / 增肌 / 维持
- **目标体重：** (必填) kg
- **目标体脂率：** (可选) %

## 热量与宏量目标

### 热量
- **每日热量目标：** kcal/天 (auto)
- **热量缺口/盈余：** kcal/天 (auto)
- **预计速度：** kg/周 (auto)

### 宏量营养素

参考范围（按目标类型）：

| 目标 | 蛋白质 | 碳水 | 脂肪 |
|------|--------|------|------|
| 减脂 | 2.0-2.4g/kg* | 2-4g/kg | 剩余热量 |
| 增肌 | 1.6-2.2g/kg | 4-6g/kg | 剩余热量 |
| 维持 | 1.6-2.0g/kg | 3-5g/kg | 剩余热量 |

> *肥胖用户（BMI>30）蛋白质按**瘦体重**计算（1.2-1.6g/kg lean mass），而非体重。
> 如无体脂率数据，按体重 × 1.6g 估算。

- **蛋白质目标：** g/天
- **碳水目标：** g/天
- **脂肪目标：** g/天
- **纤维目标：** g/天 (≥25g)

### 水分
- **每日饮水目标：** L/天 (基线: 35ml/kg 体重)

## 设置

- **exercise_counts_as_budget:** false
  > 运动消耗是否计入剩余热量预算。
  > `false`（默认）：剩余 = 目标 - 已摄入。运动消耗单独展示但不增加可吃额度。
  > `true`：剩余 = 目标 - 已摄入 + 运动消耗。更灵活但容易导致"补偿心理"。
